В основе декаратора лежит абстрактный класс.

Основная реализация расширяет асбтрактный класс, переопределает абстрактные методы
и готов к исполнению.

Для того, чтоб расширить испольняющий класс не затрагивая остальные реализации, но
не просто расширить, а расширить переиспользуюемый код, создается ещё суб абстрактный класс,
который расширяет первый абстрактный класс.
Все переиспользуемые куски кода, расшираяют суб абстрактный класс, и переопредляют абстрактные методы
двух асбтрактных классов. А так же каждый объект расширяющий субасбтрактный класс, должен иметь
конструктор, который принимает инстанс, объекта расширяющего инстанса асбтрактоного класса(мы должны
знать, что расширяем). Это позваляет расширять объект переиспользуемыми кусками кода.

Далее идет магия:
Вначале мы создаем инстанс основной реализации

АбстрактныйКласс абсК = new АбстрактныйКлассОС();//АбстрактныйКлассОсновнаяРеализация

далее мы оборочиваем основную реализацию, в расщиряющий класс.

абсК = new SubАбстрактныйКласс(абсК)

теперь в абсК, появилась новая реализация или измененная реализация тех или иных методов.
